# Corso Git - Version Control System
Questo corso introduce i concetti fondamentali del controllo di versione con Git, partendo dalle basi teoriche fino ad arrivare a branching avanzato, collaborazione e workflow professionali.

Il corso è costruito attorno a **un progetto reale**, che evolve modulo dopo modulo, mantenendo sempre un filo logico tra teoria e pratica.

Non è un corso di soli comandi: l'obiettivo è **imparare a ragionare con il version control**, comprendendo la logica interna di Git e sviluppando un metodo di lavoro solido e riproducibile.

---

## Idea centrale del corso

Tutto il corso ruota attorno a **un progetto concreto** che evolve nel tempo.

Non studiamo Git "per comandi isolati", ma perché:

- emerge un problema reale di gestione del codice
- serve uno strumento specifico
- lo applichiamo subito al progetto

Questo garantisce:

- continuità logica
- coerenza tra le lezioni
- competenze realmente spendibili in contesti professionali

---

## Obiettivi del corso

- comprendere il modello interno di Git (staging, tree, commits)
- saper gestire repository locali e remoti
- padroneggiare branching e merging in scenari reali
- risolvere conflitti in modo consapevole
- lavorare in team con workflow collaudati (Git Flow, GitHub Flow)
- utilizzare strumenti collaborativi (pull request, code review)

---

## Struttura del corso

- **Progetto**: costruzione incrementale di un repository reale
- **Moduli**: concetti teorici + applicazione pratica immediata
- **Esercizi**: approfondimento guidato, non verifica
- **Soluzioni community**: le migliori spiegazioni vengono integrate

---

## Partecipazione della community

Gli studenti possono:

- proporre esercizi
- migliorare spiegazioni
- suggerire soluzioni alternative
- documentare casi d'uso reali

Ogni modifica passa tramite Pull Request e viene approvata secondo la regola **50% + 1**.

---

## Tecnologie utilizzate

- Git (versione 2.x o superiore)
- GitHub / GitLab come piattaforme di hosting remoto
- CLI come interfaccia principale
